The Standard Toolkit

The standard toolkit is the collection of methods that are built into Sequence. The standard toolkit provides the methods for scripting logic such as branching (if-else-endif), looping (begin-while-repeat), and working with the stack, and data methods for loading/storing/deleting variables, and JSON data.

Stack Manipulation

These methods are used to perform actions on the stack. They include operations such as duplicating items, reordering items, droping items, etc.

{ "op": "dup", ... }

Duplicates the item at the top of the stack. There are parameters to control if a shallow or deep copy is done, as well as control the TOS offset of the item to be duplicated.

Parameters

deep: bool(default: false)

Controls if the copy is a shallow or deep copy.

offset: int(default: 0)

The offset (from TOS) of the item you want to duplicate.

Outputs

item_copy: Any

A copy of the requested item.

{ "op": "swap", ... }

Swaps the order of the items at the top of the stack.

Parameters

order: list(default: [1, 0])

The new order of the TOS. The first integer in the list is TOS offset of the item that should be moved to the TOS. The length of the list controls the number of items that are reordered.

{ "op": "drop" }

Drops the item at the top of the stack.

{ "op": "size" }

Returns the current size (depth) of the stack.

Outputs

stack_size: int

The size of the stack.

{ "op": "pack", ... }

Packs N items from the top of the stack into a single array at the top of the stack.

Parameters

n*: int (*required)

The number of items at the top of the stack to pack into the array.

forward: bool(default: true)

If false, the items are packed in reverse order.

keys: list(default: None)

If provided, the items are packed as key-value pairs. The last key applies to the TOS item.

Inputs

...: Any

n items are consumed from the top of the stack.

Outputs

arr: Array

An array of items.

{ "op": "unpack", ... }

Packs N items from the top of the stack into a single array at the top of the stack.

Parameters

keys: list(default: None)

Required to unpack a list of key-value pairs. The last key refers to the new TOS item.

Outputs

...: Any

The unpacked items.

Looping

These methods facilitate looping and recursion. Loops start with begin and end with repeat. Loops are terminated by break. The while method is provided for convenience, and effectively guards a break in an if-block.

{ "op": "begin" }

Marks the beginning of a loop.

{ "op": "while" }

Continues the loop if the item at the top of the stack is true. If the item at the top of the stack is not true, it terminates the loop.

Inputs

cond: bool, Any

If true, the loop continues. If false, the loop terminates.

{ "op": "foreach" }

Loops through each element of an iterable. The next element is placed at the TOS at the start of each loop.

Inputs

[iterable]: list, iterable

The thing that is iterated through. This input only applies to initializing the loop.

Outputs

element: Any

The next element.

{ "op": "break" }

Breaks out of a loop (terminates the loop).

{ "op": "repeat" }

Marks the end of a loop.

{ "op": "recurse" }

Restarts the current sequence.

Branching

These methods facilitate conditional branching.

{ "op": "if" }

Marks the beginning of an if-block. An if statment MUST be terminated by `{"op": "endif"}`.

Inputs

cond: bool, Any

If true, the if-block is evaluated. If false, the else-block is evaluated if it exists.

{ "op": "else" }

Marks the beginning of an else-block. This method MUST be placed between `{"op": "if"}` and `{"op": "endif"}`.

{ "op": "endif" }

Marks the end of an if statement.

{ "op": "asfloat" }

Outputs

Arithmetic

These methods are used to perform arithmetic.

{ "op": "/", ... }

Divides the two numbers at the top of the stack. If the items at the top of the stack are not numbers, the binary operator '/' is applied to the objects.

Parameters

value: Any(default: None)

Used to specify y as a parameter. Disables input y when specified.

div0_result: float(default: None)

If div0_result ir provided and y is zero, result is set to div0_result.

Inputs

[y]: number, Any

The denominator.

[x]: number, Any

The numerator.

Outputs

result: number, Any

The result of the division.

{ "op": "*", ... }

Inputs

[y]: number, Any

The second term.

[x]: number, Any

The first term.

Outputs

result: number, Any

The result of the multiplication.

Miscellaneous

These methods don't fit into the other categories, but they are useful.

{ "op": "fstring", ... }

Formats a string.

Parameters

fmt*: str (*required)

The format string (a python f-string). See the Python documentation of f-strings details.

**params: None(default: None)

Named arguments for the f-string.

Inputs

...: Any

The top N items are poped from the stack where N is the number of positional arguments in the f-string.

Outputs

result: str

The formatted string.

{ "op": "assert", ... }

Asserts that the item at the top of the stack is true. Terminates execution if false.

Parameters

error: str(default: None)

The error message to print if the assertion fails.

negate: bool(default: false)

Assert that the item at the top of the stack if false (rather than true).

Inputs

cond: bool

The boolean value to check.

Data Methods

The standard toolkit provides data methods for loading/storing/deleting local variables, and JSON data.

Variables

These methods are used to store, load, and delete variables. The scope of variables is local.

{ "op": "get", "uri": "variables://..." , ... }

Loads a variable and places it at the top of stack.

Parameters

uri*: str (*required)

The URI of the resource to load

default: Any(default: None)

The default value if the variable doesn't exist.

Outputs

data: Any

The variable.

{ "op": "put", "uri": "variables://..." }

Saves a variable (sequence-scope).

Parameters

uri*: str (*required)

The URI of where the resource should be saved

Inputs

data: None

The resource to save

{ "op": "del", "uri": "variables://..." }

Deletes a variable.

Parameters

uri*: str (*required)

The URI of the resource to delete
